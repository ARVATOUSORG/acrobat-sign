Getting Started¶

Note

This beta document is under construction. Send feedback and requests to brogers@adobe.com.

Recommended first steps¶

While optional, Adobe recommends:

  • Creating a backup admin manually.

  • Reviewing the system requirements.

Okta tenant setup¶

Okta authentication¶

Okta provides the authentication mechanism for Acrobat Sign, and you must have an Okta credential before gaining product access. Adobe provides customers with a unique Okta instance, and there are two options for configuring user access:

  • Use Okta directly as your identity provider. This workflow requires each user to authenticate individually. It typically results in one less login screen.

  • Integrate Okta with your organization’s existing identity provider and directory service. You can use a SAML 2.0-compliant identity provider to manage user identities in bulk. This workflow allows users to only see the Okta login page a single time if they choose “Remember Me” during login.

Once Adobe provisions the Okta account, the primary (tenant) administrator receives an email from Okta with a one-use activation link to the account’s Okta portal and the domain-claiming token needed to provision users on the Acrobat Sign system. Immediately do the following:

  1. Use the one-time activation link to log in to the Okta system and set your password. It expires in 7 days.

  2. Bookmark the Okta portal when you authenticate. Your Okta portal URL is unique to your account.

  3. Record the DNS name, admin username, and admin password.

  4. If you are using a directory service, connect it now as described HERE.

Domain claiming¶

Claiming your domains is required. The environment prevents creating users with email addresses whose domains are not explicitly claimed by the account. The DNS domain could be a subdomain of your second-level domains; for example, adobesigngov.example.org if example.org is your domain.

Prerequisites¶

  • Know your domains.

  • Have access to any inboxes for email addresses on the DNS domain under your control. These are likely inboxes directly related to your DNS domain. Alternatively, you could use email forwarding; for example, my.user@adobesigngov.example.org could forward to my.user@example.org. You must be able to read invitation/activation emails for your users sent from your Okta instance to email addresses on your DNS domain.

  • For multi-tenant partner integration accounts, it’s recommended to also have a separate DNS domain for customer testing. This allows you to test customer account usage of a partner integration that’s defined on your partner account.

Claim domain steps¶

When you receive the domain claiming token from the Okta system, add the token to DNS text records as follows:

  1. Log in to your domain account.

  2. Find the page for updating the DNS record. This page may be called DNS Management, Name Server Management, or Advanced Settings.

  3. Find the TXT records for your domain.

  4. Add a TXT record with the full token value supplied by Adobe. For example: adobe-sign-verification=f9ccbThisIsNotARealNumber832

  5. Save your changes.

Tip

For more detail, see https://helpx.adobe.com/sign/fedramp/okta/okta-claim-domains.html.

Registering an API application¶

Adobe need to validate and enable API applications and must create all apps. You will therefore work either with Adobe directly or with your partner during this process. The steps include:

  1. Complete the Okta tenant setup.

  2. Ask Adobe to register a customer (single-tenant) or partner (multi-tenant) API app.

  3. Provide Adobe with the following:

  • A redirect URL which links the app instance to the correct Acrobat Sign account.

  • A list of required scopes that define what resources and actions are available to your app.

  • An app name.

  • Whether or not you require impersonation.

  1. Adobe registers the app.

  2. Adobe sends an encrypted PDF with the client_id and client_secret.

  3. A Professional Services team member calls with a password for the encrypted PDF.

You can now start using the APIs in your app.
